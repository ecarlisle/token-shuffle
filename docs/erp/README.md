# Engineering Review Packets

Engineering Review Packets, or ERPs, are actionable work items created from Engineering Reviews.

ADRs record decisions.
ERPs record review observations and proposed improvements.

## Purpose

ERPs preserve the reasoning behind improvement work so feedback does not disappear into chat, issue trackers, or pull request comments.

Each ERP should describe:

- what was observed
- why it matters
- what should change
- how completion will be verified
- which files are likely affected
- how the work was resolved

## Status Values

- Proposed
- Accepted
- In Progress
- Completed
- Deferred
- Superseded
