# v0.6 release validation

v0.6 adds Anthropic Messages ingress without flattening it into OpenAI wire
semantics.

Automated evidence verifies:

- raw Anthropic message/content/tool blocks and unknown fields reach the
  Anthropic target unchanged;
- a local bearer or Anthropic-style `x-api-key` is consumed and replaced by the
  provider `x-api-key` and configured `anthropic-version`;
- buffered bodies, provider errors, and Anthropic SSE event bytes/order are
  preserved;
- Anthropic input/output and cache-read/cache-creation usage become
  provider-reported domain schema-v2 events carrying `anthropic-messages` and
  `anthropic` identity;
- existing schema-v1 OpenAI events remain readable;
- OpenAI and Anthropic adapters coexist and are selected by ingress capability;
- neither adapter retries or fails over to the other.

Release commands:

```sh
pnpm install --frozen-lockfile
pnpm check
pnpm build
pnpm test
pnpm --filter @token-shuffle/web test:e2e
```
