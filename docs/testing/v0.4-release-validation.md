# v0.4 release validation

v0.4 validates deterministic conversation compaction against a synthetic,
privacy-safe long-session corpus.

## Retention contract

The replay fixtures require the forwarded request to retain:

- all system and developer messages verbatim;
- the configured active message window verbatim;
- tool-call/result boundaries at the active-window edge;
- objectives and completion requirements;
- explicit constraints and prohibitions;
- file paths and symbols;
- changed-artifact statements;
- failures and errors;
- accepted decisions;
- open questions;
- an uncertainty statement.

Every applied summary records its source start/end indexes, deterministic
fingerprint, and `deterministic-v1` summary version. Changing source material
must change the fingerprint.

## Accounting and failure contract

Deterministic summary construction consumes zero model input/output tokens and
creates no extra provider attempt. Literal and net reduction are measured once
from the original request to the final prepared request.

Compaction preserves the original request when:

- the policy or global policy system is disabled;
- the message threshold is not reached;
- no eligible old turns exist;
- source characters exceed the configured bound;
- the structured candidate is not smaller.

Applied compaction keeps an eight-hour, bounded memory-only source snapshot. The
administrative recovery endpoint requires the separate dashboard session,
validates the fingerprint, sets `no-store`, and returns `404` after expiry or
restart. Recovery content never enters the event ledger.

The fake-provider suite verifies one upstream attempt and inspects the exact
prepared request and policy event. The domain replay suite verifies positive net
reduction across the complete synthetic session.

## Release command

```sh
pnpm check
pnpm build
pnpm --filter @token-shuffle/web test:e2e
```

Live optimize-mode client compatibility remains provisional. Persistent
artifacts and model-directed retrieval remain v0.5 scope.

## v0.4.1 patch validation

The v0.4.1 patch adds no new token-reduction policy. It validates:

- frozen-install workspace resolution from a checkout without `dist` or
  `node_modules`;
- preservation of an existing v0.4 SQLite event database through versioned
  migration startup;
- default byte-preserving provider dispatch;
- explicitly configured DeepSeek-compatible `developer` to `system` role
  normalization at the provider boundary;
- measurement of the original developer-message count and final outbound byte
  boundary;
- retention of supported roles, message fields, and unrelated request fields.

Release verification uses Node.js 24.15+ and runs:

```sh
pnpm install --frozen-lockfile
pnpm check
pnpm build
pnpm test
pnpm --filter @token-shuffle/web test:e2e
```

The provider normalization remains provisional pending a repeated live
OpenCode/DeepSeek smoke test.

Validation result on 2026-07-02:

- Node.js 24.15.0 and pnpm 11.7.0;
- frozen install from a temporary checkout without `node_modules` or `dist`;
- 9 core, 3 web unit, and 35 proxy tests passed;
- all workspace production builds passed;
- 2 Playwright dashboard journeys passed;
- no live-provider smoke test was claimed.
