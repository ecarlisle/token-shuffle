# v0.5 release validation

v0.5 validates local externalized context and explicit next-turn retrieval
without introducing hidden provider retries.

## Privacy and migration contract

- Artifact IDs and query identities are HMAC-SHA-256 values scoped by an
  environment-resolved installation key.
- Retrieval is disabled by default.
- Enabling retrieval visibly retains eligible readable context for seven days
  by default, separately from structural events and diagnostic prompt/response
  capture.
- Schema migration 0002 upgrades a real v0.4 database transactionally.
- Request, session, history deletion, and expiry remove associated artifacts.
- Credentials, keys, queries, and artifact content never enter observation
  events.

## Selection and recovery contract

- Persistence and search require and are limited to an explicit session ID.
- An exact artifact identifier is checked before FTS5 lexical matching.
- Results are bounded by count and injected characters.
- `token_shuffle_retrieve("query")` is recognized only when present in an
  inbound turn; a model response is therefore fulfilled on the next normal
  client request.
- A miss and a storage failure both preserve the request and issue one provider
  attempt; evidence distinguishes them and records `retryCount: 0`.
- Same-turn tool interception and semantic embeddings are not claimed.

## Accounting evidence

The accepted two-request fixture:

1. compacts an old conversation and externalizes its original source;
2. retrieves that artifact by exact ID on the same explicit session;
3. caps injected source at 512 characters;
4. records two requests and two total provider attempts;
5. remains net-positive when baseline minus final forwarded input is summed
   across the complete session.

## Release commands

Run with Node.js 24.15+:

```sh
pnpm install --frozen-lockfile
pnpm check
pnpm build
pnpm test
pnpm --filter @token-shuffle/web test:e2e
```

Live OpenCode/Pi retrieval compatibility remains provisional. The release does
not claim that every client automatically continues after a model emits the
retrieval marker.

Validation result on 2026-07-02:

- Node.js 24.15.0 and pnpm 11.7.0;
- frozen install completed without lockfile changes;
- 10 core, 3 web unit, and 40 proxy tests passed;
- all workspace production builds passed;
- 2 Playwright dashboard journeys passed;
- no live-provider retrieval smoke test was claimed.
