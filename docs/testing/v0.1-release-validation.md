# v0.1 release validation

This checklist records the promotion of `v0.1.0-rc.1` to v0.1.0 and remains the
template for expanding verified compatibility. Synthetic protocol tests are
necessary but do not replace live checks.

## v0.1.0 result

Released on 2026-06-30 after a live OpenCode 1.17.11 session through OpenCode
Zen using `deepseek-v4-flash-free`. The event ledger recorded eight complete
request lifecycles with no degraded persistence. Pi 0.79.8 and OpenAI direct
remain provisional because no equivalent proxied session was recorded.

## Record the test matrix

For each run, record:

- Token Shuffle commit and version;
- operating system and Node.js version;
- client name and exact version;
- upstream provider, model ID, and date;
- buffered, streaming, tool-call, cancellation, and provider-error results.

Never record API keys, prompts containing private code, or readable response
content.

## OpenAI reference upstream

1. Set `upstream.baseUrl` to `https://api.openai.com/v1`.
2. Reference the provider credential through `fromEnv`.
3. Run `token-shuffle config validate`.
4. Run `token-shuffle doctor`; require successful SQLite, TLS, authentication,
   and model-discovery checks.
5. Start Token Shuffle and confirm authenticated `status` reports
   `persistence.degraded: false`.

Use a currently available tool-capable model and record its exact identifier
rather than embedding a moving model alias in this checklist.

## Client sessions

Run the documented [OpenCode](../getting-started/agents/opencode.md) and
[Pi](../getting-started/agents/pi.md) configurations against the same pinned
model. For each client:

1. complete a read-only buffered request where supported;
2. complete a streamed response and verify the terminal event;
3. invoke a harmless tool and return its result;
4. exercise parallel tool calls when supported by the model/client;
5. cancel an active stream and confirm upstream work stops;
6. induce a safe provider error and confirm status/body fidelity;
7. confirm local authorization and internal session headers never reach the
   provider;
8. confirm persisted events contain structural metrics and no readable prompt,
   response, or credential content.

## Promotion gate

Promote a client/upstream combination to **Verified** only when:

- all automated checks pass under Node.js 24.15+;
- at least one live client session passes against that upstream;
- the [compatibility matrix](../compatibility.md) names tested versions,
  provider, model, date, and feature results;
- no test required disabling authentication, retention, limits, or fidelity
  protections.
